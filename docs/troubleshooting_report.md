# RVR 设备数据异常排查分析报告

## 1.现象描述
* 故障现象：
RVR 间歇性掉数据，终端面板间歇性显示红色斜杠。
*  背景信息：机场设备现场，RVR终端显示系统
## 2.异常排查逻辑
本分析旨在验证“数据传输是否存在完全中断”。

* **假设 1：** 掉包表现为时间戳不连续。
* **数据源：** 2026-06-26 日期范围日志。
* **计算逻辑：** 计算相邻记录 `CREATEDATE`的`time_diff`。
## 3.排查结论（待验证）
* **观察结果：** 在2026练6月26日00：00 至23：59的数据集中，未发现`time_diff>15s`的记录。
* **风险提示：** 
    * **数据传输连续性良好，不代表硬件链路无故障。**
    * 需考虑“物理链路处于临界状态”的可能性：虽物理信号未中断，但可能存在高误码率导致设备自动重发或丢包。
    * **当前仅排除了“物理链路完全中断”的可能。**
## 4.下一步排查方向
* **方向A：数值异常探测**
    * **逻辑：** 虽然时序连续，但是特定时刻的脏数据，可能是底层采样电路抖动的信号。
    * **执行操作：** 统计RVR_1A及MOR_1A等核心指标的离群点，分析其于数据丢失的相关性。
* **方向B：系统状态分析**
    * **逻辑：** 检查`STATUS`列是否存在非正常值，这可能时通信临界故障的证据。
    * **执行操作：** 提取所有`STATUS`的非正常值。
* **方向C：环境相关性验证**
    * **逻辑：** 数据采集、传输故障可能与环境对设备的影响（如温度、湿度变化）有关
    * **执行操作：**  获取同时间段的环境参数，与RVR数据波动进行关联分析。
